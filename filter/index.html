Sélection conditionnelle

En SQL, il est possible de sélectionner certaines lignes dans notre table selon des conditions. L’instruction WHERE permet de filtrer nos données selon les conditions que l’on renseigne.

SELECT colonne 
FROM tableau 
WHERE colonne opérateur condition

On teste une condition à l’aide des opérateurs de comparaisons.

=  # Égal
<> # Différent de
!= # Différent de
>  # Strictement supérieur
<  # Strictement inférieur
>= # Supérieur ou égal
<= # Inférieur ou égal

 

SELECT *
FROM tableau 
WHERE taille > 178

Sélection conditionnelle

Le langage SQL propose d’autres opérateurs pour réaliser des conditions.

Parmi eux, les plus utilisés :

  • (NOT) IN : Pour tester une liste de plusieurs valeurs.

  • (NOT) BETWEEN : Pour tester une valeur au sein d’un intervalle.

  • (NOT) LIKE : Effectuer une recherche sur un mot.

  • IS NULL : Pour tester si une valeur est nulle.

  • IS NOT NULL : Pour tester si une valeur est non nulle.

IN

L’opérateur IN

L’opérateur IN vérifie si les valeurs d’une colonne sont égales à une des valeurs dans une liste donnée.

L’opérateur IN permet d’éviter la répétition de l’opérateur OR

SELECT colonne
FROM tableau
WHERE colonne IN (valeur_1, valeur_2, .., valeur_x)

En deux requêtes, une avec des opérateurs OR et l’autre avec l’opérateur IN :

Extraire tous les joueurs évoluant dans les équipes nationales de France, Belgique, Espagne et Russie.

L’opérateur IN

Avec l’opérateur OR

dbGetQuery(
  conn = db, 
  "SELECT * 
  FROM players 
  WHERE team = 'France'
    OR team = 'Belgique' 
    OR team = 'Espagne' 
    OR team = 'Russie'"
)

Avec l’opérateur IN

dbGetQuery(
  conn = db, 
  "SELECT * 
  FROM players
  WHERE team IN ('France', 'Belgique', 'Espagne', 'Russie')"
)

On préfère avec l’opérateur IN ! 😉

BETWEEN

L’opérateur BETWEEN

L’opérateur BETWEEN permet de filtrer dans un intervalle de données.

L’intervalle peut être constitué de chaînes de caractères, de nombres ou de dates.

Le plus souvent, on l’utilise sur les deux derniers formats : les nombres et les dates.

SELECT colonne
FROM tableau
WHERE colonne BETWEEN valeur_1 AND valeur_2

Pour tester si une valeur n’intervient pas dans un intervalle, il faut écrire NOT BETWEEN.

L’opérateur BETWEEN

Extraire les joueurs nés entre le 01 janvier 1990 (1990-01-01) et le 31 décembre 1994 (1994-12-31)

Extraction des joueurs nés entre le 01 janvier 1990 (1990-01-01) et le 31 décembre 1994 (1994-12-31)

  FROM players

LIKE

L’opérateur LIKE

L’opérateur LIKE permet de filtrer une colonne selon un pattern donné. On l’utilise pour effectuer des recherches sur des mots, par exemple.

SELECT *
FROM tableau
WHERE colonne LIKE pattern

A l’inverse de l’opérateur de comparaison =, l’opérateur LIKE est plus maniable, notamment grâce à ces deux jokers :

  • % : remplace tous les autres caractères.

  • _ : remplace un et uniquement un caractère.

Vite, vite, voyons quelques exemples !

L’opérateur LIKE

Sélectionnons tous les joueurs dont les noms commencent par la lettre “a” :

dbGetQuery(

  FROM players 
  WHERE name LIKE 'a%'"
)

Sélectionnons tous les joueurs dont les noms terminent par la lettre “a” :

dbGetQuery(

  "SELECT * 
  FROM players 
  WHERE name LIKE '%a'"
)

Sélectionnons tous les joueurs dont les noms contiennent la lettre “a” et également au moins un “c” :

dbGetQuery(
  conn = db,
  "SELECT * 
  FROM players 
  WHERE name LIKE '%a%c%'"
)

IS (NOT) NULL

Les opérateurs IS NULL / IS NOT NULL

L’opérateur IS permet de tester si une valeur est NULL ou non, NOT NULL.

C’est la seule façon de gérer les valeurs manquantes. Les valeurs manquantes ne sont pas gérées par les autres opérateurs de comparaisons.

Mettons en pratiques ces deux opérateurs sur notre table !

Sélectionner les joueurs dont la position sur le terrain est inconnue :

dbGetQuery(
  conn = db, 
  "?"
)

Sélectionnons les joueurs dont la position sur le terrain est inconnue :

dbGetQuery(
  conn = db, 
  "SELECT * 
  FROM players 
  WHERE position IS NULL"
)

ORDER BY

En SQL, il est possible de gérer dans notre requête, l’ordre de sortie des résultats.

À l’aide de l’instruction ORDER BY, on peut trier les lignes d’une ou plusieurs colonnes par ordre croissant ou décroissant.

À noter, par défaut, l'ordre d'affichage est toujours croissant !

SELECT *
FROM tableau
ORDER BY colonne DESC

Pour trier sur plusieurs colonnes, on les nomme dans l’instruction ORDER BY et on les sépare par des ,
